A transaction request describes what a wallet should ask the user to approve.

## Request parts

| Part        | Meaning                              |
| ----------- | ------------------------------------ |
| Recipient   | Destination account.                 |
| Amount      | Value to send.                       |
| Payload     | Optional application data.           |
| State init  | Optional contract deployment data.   |
| Valid until | Expiration time for wallet approval. |

## Wallet approval

Wallet approval means the wallet accepted the request. It does not prove final chain inclusion.

## Production rule

Track and verify transactions before delivering paid goods.

A session lets a connector restore a wallet connection after a page reload.

## Session states

| State        | Meaning                                                  |
| ------------ | -------------------------------------------------------- |
| New          | No wallet has been connected.                            |
| Connected    | A connector has an active wallet session.                |
| Restored     | Stored connector state was loaded on startup.            |
| Expired      | Stored state exists but the wallet no longer accepts it. |
| Disconnected | The application or user ended the session.               |

## Rule

Treat restore as a best effort. Always handle expired or missing sessions.

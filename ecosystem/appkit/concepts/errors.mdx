AppKit flows can fail at the application, connector, wallet, provider, or chain layer.

## Common classes

| Class         | Example                                | Recovery                                    |
| ------------- | -------------------------------------- | ------------------------------------------- |
| User action   | User rejects a wallet request.         | Keep state unchanged and allow retry.       |
| Configuration | Missing connector or network.          | Fix AppKit config before showing the flow.  |
| Provider      | RPC or quote provider fails.           | Show retry and fallback state.              |
| Request       | Malformed address, amount, or payload. | Validate before opening wallet.             |
| Chain         | Transaction is not found or not final. | Continue tracking or ask backend to verify. |

## Rule

Rejection is not exceptional. It is a normal wallet path.

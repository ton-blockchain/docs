Toncoin is the native asset on TON. AppKit supports balance reads, transfer request creation, wallet submission, and settlement tracking. All actions take the `AppKit` instance as the first argument.

Amounts are passed as decimal strings in human-readable units. Toncoin uses nine decimal places, so floating-point math can lose precision. Keep amounts as strings until you render them.

## Read balance

`getBalance` reads the selected wallet balance once. `watchBalance` subscribes to balance changes and returns an unsubscribe function. Both actions have `*ByAddress` variants for reading an arbitrary address instead of the selected wallet.

```ts
import { getBalance, watchBalance } from '@ton/appkit';

const ton = await getBalance(appKit);
// "12.345678901"

const stop = watchBalance(appKit, {
  onChange: (update) => {
    // update.balance (formatted string), update.rawBalance (nanos), update.address
  },
});
// Call stop() when done.
```

## Build and send a transfer

A transfer can be handled in two steps: build a `TransactionRequest`, then pass it to `sendTransaction`. This lets an app inspect or modify the request before the wallet signs it, for example to attach a custom payload.

```ts
import { createTransferTonTransaction, sendTransaction } from '@ton/appkit';

const request = createTransferTonTransaction(appKit, {
  recipientAddress: 'EQ...',
  amount: '1.25',
  comment: 'Order #1001',
});
const response = await sendTransaction(appKit, request);
// response.boc, response.normalizedHash
```

| Parameter           | Required | Meaning                                                                       |
| ------------------- | -------- | ----------------------------------------------------------------------------- |
| `recipientAddress`  | Yes      | Destination address.                                                          |
| `amount`            | Yes      | Amount in TON, as a decimal string. Converted to nanos internally.            |
| `comment`           | No       | Free-text comment. Encoded as a text payload.                                 |
| `payload`           | No       | Base64 BoC. When present, takes priority over `comment`.                      |
| `stateInit`         | No       | Base64 state init for deploying the destination contract on the same message. |

When you do not need to inspect the request, `transferTon` builds and sends it in one call:

```ts
import { transferTon } from '@ton/appkit';

const response = await transferTon(appKit, {
  recipientAddress: 'EQ...',
  amount: '1.25',
});
```

Both `createTransferTonTransaction` and `transferTon` throw when no wallet is selected.

## Track settlement

A wallet response with a `boc` means the user signed and submitted the transaction. The transaction still has to land on chain. `getTransactionStatus` performs a one-shot trace lookup, while `watchTransactions` subscribes to the selected wallet's transaction stream.

```ts
import { getTransactionStatus } from '@ton/appkit';

const status = await getTransactionStatus(appKit, { boc: response.boc });
if (status.status === 'completed') {
  // Confirmed on chain.
}
```

`status` is `'unknown' | 'pending' | 'completed' | 'failed'`. The response also includes `totalMessages`, `pendingMessages`, and `onchainMessages` for trace progress. Pass either `boc` (recommended — the action computes the normalized hash) or `normalizedHash` directly.

## Rules

- Validate the recipient address, amount, and expected network before calling any mutation. Do not rely on the wallet as the final input validator.
- Treat `sendTransaction` returning a `boc` as "user signed", not "settled". Wait for `getTransactionStatus` to return `completed` before delivering value.
- Refetch wallet balances and cached lists after settlement. Use trace status, not the immediate wallet response, as the trigger for product state changes.
- Do not derive product success from the wallet response shape. Verify on the chain.
- Keep amounts as strings until you display them. Floating-point math loses precision at TON's nine decimal places.

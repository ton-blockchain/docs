Toncoin is the native asset on TON. AppKit covers reading balances, building transfer requests, sending them through the connected wallet, and tracking settlement.

All actions take the `AppKit` instance as the first argument. The two-step pattern — `create*Transaction` + `sendTransaction` — is exposed as a single convenience function (`transferTon`) when you don't need to inspect the request first.

## Read balance

| Action                  | Returns                                                  |
| ----------------------- | -------------------------------------------------------- |
| `getBalance`            | `Promise<TokenAmount \| null>` for the selected wallet.  |
| `getBalanceByAddress`   | `Promise<TokenAmount>` for any address.                  |
| `watchBalance`          | Subscribes to the selected wallet's balance.             |
| `watchBalanceByAddress` | Subscribes to any address's balance.                     |

`TokenAmount` is a string in human-readable units. Toncoin uses 9 decimal places.

```ts
import { getBalance, watchBalance } from '@ton/appkit';

const ton = await getBalance(appKit);
// "12.345678901"

const stop = watchBalance(appKit, {
  onChange: (update) => {
    // update.balance (formatted string), update.rawBalance (nanos), update.address
  },
});
// Call stop() when done.
```

## Build a transfer

`createTransferTonTransaction` produces an unsigned `TransactionRequest`. Pass it to `sendTransaction` to ask the connected wallet to sign and broadcast.

```ts
import { createTransferTonTransaction, sendTransaction } from '@ton/appkit';

const request = createTransferTonTransaction(appKit, {
  recipientAddress: 'EQ...',
  amount: '1.25',
  comment: 'Order #1001',
});
const response = await sendTransaction(appKit, request);
// response.boc, response.normalizedHash
```

| Parameter           | Required | Meaning                                                                                  |
| ------------------- | -------- | ---------------------------------------------------------------------------------------- |
| `recipientAddress`  | Yes      | Destination address.                                                                     |
| `amount`            | Yes      | Amount in TON, as a decimal string. Converted to nanos internally.                       |
| `comment`           | No       | Free-text comment. Encoded as a text payload.                                            |
| `payload`           | No       | Base64 BoC. When present, takes priority over `comment`.                                 |
| `stateInit`         | No       | Base64 state init for deploying the destination contract on the same message.            |

`createTransferTonTransaction` throws if no wallet is selected.

## One-shot transfer

`transferTon` chains build and send.

```ts
import { transferTon } from '@ton/appkit';

const response = await transferTon(appKit, {
  recipientAddress: 'EQ...',
  amount: '1.25',
});
```

## Track settlement

A wallet returning a `boc` only means the user signed. Wait for the chain to confirm.

| Action                         | Returns                                                        |
| ------------------------------ | -------------------------------------------------------------- |
| `getTransactionStatus`         | `Promise<TransactionStatusResponse>`. One-shot trace lookup.   |
| `watchTransactions`            | Subscribes to selected wallet transactions.                    |
| `watchTransactionsByAddress`   | Subscribes to a specific address.                              |

`TransactionStatusResponse.status` is `'pending' | 'completed' | 'failed'`. Use `completedMessages` and `totalMessages` to render progress.

```ts
import { getTransactionStatus } from '@ton/appkit';

const status = await getTransactionStatus(appKit, { boc: response.boc });
if (status.status === 'completed') {
  // safe to deliver value
}
```

You can pass either `boc` (recommended; the action computes the normalized hash) or `normalizedHash` directly.

## React hooks

| Hook                         | Wraps                          |
| ---------------------------- | ------------------------------ |
| `useBalance`                 | `getBalance`                   |
| `useBalanceByAddress`        | `getBalanceByAddress`          |
| `useWatchBalance`            | `watchBalance`                 |
| `useWatchBalanceByAddress`   | `watchBalanceByAddress`        |
| `useSendTransaction`         | `sendTransaction`              |
| `useTransferTon`             | `transferTon`                  |
| `useTransactionStatus`       | `getTransactionStatus`         |
| `useWatchTransactions`       | `watchTransactions`            |
| `useWatchTransactionsByAddress` | `watchTransactionsByAddress` |

```tsx
import { useTransferTon, useTransactionStatus } from '@ton/appkit-react';

function PayButton() {
  const transferTon = useTransferTon();
  return (
    <button
      onClick={() =>
        transferTon.mutate({
          recipientAddress: 'EQ...',
          amount: '1.25',
          comment: 'Order #1001',
        })
      }
    >
      Pay 1.25 TON
    </button>
  );
}

function Confirmation({ boc }: { boc: string }) {
  const { data } = useTransactionStatus({ boc });
  return <span>{data?.status ?? 'pending'}</span>;
}
```

Watcher hooks return `void`. They keep the matching query cache fresh while mounted, so a paired `useBalance` will live-update without further wiring.

## Failure modes

| Class           | Example                                | Recovery                                       |
| --------------- | -------------------------------------- | ---------------------------------------------- |
| User rejected   | User dismisses the wallet prompt.      | Keep state unchanged. Allow retry.             |
| Bad input       | Invalid recipient address.             | Validate before opening the wallet.            |
| Insufficient    | Wallet does not have enough TON.       | Read balance first; show a top-up prompt.      |
| Provider        | API client or trace API is unreachable.| Retry with backoff. Show a connectivity hint.  |
| Stuck pending   | Trace returns `pending` for too long.  | Continue polling. Surface a long-pending UI.   |

## Rules

- Validate the recipient address and amount before calling any mutation. Wallets should not be the input validator of last resort.
- Treat `sendTransaction` returning a `boc` as "user signed", not "settled". Wait for `getTransactionStatus` to return `completed` before delivering value.
- Do not derive product success from the wallet response shape. Verify on the chain.
- Keep amounts as strings until you display them. Floating-point math loses precision at TON's nine decimal places.

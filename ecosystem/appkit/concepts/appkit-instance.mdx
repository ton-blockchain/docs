An AppKit instance stores configuration, connector state, selected wallet state, and provider access for one application runtime.

## Lifecycle

```text
configure -> initialize -> connect wallet -> perform actions -> observe state -> disconnect or restore session
```

## What it owns

| State      | Description                                                |
| ---------- | ---------------------------------------------------------- |
| Networks   | Mainnet, testnet, and API client configuration.            |
| Connectors | Wallet connection mechanisms.                              |
| Wallets    | Connected wallets and selected wallet state.               |
| Providers  | Data, swap, staking, or streaming integrations.            |
| Managers   | Domain-specific helpers such as swap and staking managers. |

## Rule

Create one AppKit instance for one app runtime. Pass it to actions directly or provide it to React with `AppKitProvider`.

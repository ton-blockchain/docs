NFTs on TON are individual contracts, optionally grouped into a collection. AppKit covers reading the items owned by an address, fetching one item by address, and building transfer requests. As with other assets, transfers go through the connected wallet and settle on chain.

## NFT shape

```ts
interface NFT {
  address: string;
  index: bigint;
  collection?: { address: string; name?: string; symbol?: string; image?: string };
  owner?: Address;
  metadata?: {
    name?: string;
    description?: string;
    image?: string;
    contentUrl?: string;
    attributes?: Array<{ trait_type: string; value: string }>;
    externalUrl?: string;
  };
}
```

`NFTsResponse` wraps a list with the queried address and pagination info: `{ nfts: NFT[]; address: string; pagination?: { limit; offset } }`.

## Read NFTs

| Action            | Returns                                                                        |
| ----------------- | ------------------------------------------------------------------------------ |
| `getNfts`         | `Promise<NFTsResponse \| null>` — every NFT owned by the selected wallet.      |
| `getNftsByAddress`| `Promise<NFTsResponse>` — every NFT owned by any address. Supports pagination. |
| `getNft`          | `Promise<NFT \| null>` — a single NFT contract, by address.                    |

```ts
import { getNfts, getNft } from '@ton/appkit';

const owned = await getNfts(appKit);
// owned.nfts = [{ address, index, collection, owner, metadata }]

const one = await getNft(appKit, { address: 'EQ-NFT-CONTRACT' });
```

`getNfts` returns `null` when no wallet is selected. `getNft` returns `null` when the contract is not found.

## Build a transfer

`createTransferNftTransaction` builds the NFT transfer payload, sets the response destination back to the sender, and wraps it in a `TransactionRequest` with a default gas allowance.

```ts
import { createTransferNftTransaction, sendTransaction } from '@ton/appkit';

const request = await createTransferNftTransaction(appKit, {
  nftAddress: 'EQ-NFT-CONTRACT',
  recipientAddress: 'EQ-NEW-OWNER',
  comment: 'Gift',
});
const response = await sendTransaction(appKit, request);
```

| Parameter           | Required | Meaning                                                                       |
| ------------------- | -------- | ----------------------------------------------------------------------------- |
| `nftAddress`        | Yes      | NFT item contract address.                                                    |
| `recipientAddress`  | Yes      | New owner address.                                                            |
| `amount`            | No       | Gas budget in TON. Defaults to a safe gas allowance.                          |
| `comment`           | No       | Optional comment payload attached to the transfer.                            |

The function throws if no wallet is selected. It is async because it composes the transfer body off the NFT contract.

## One-shot transfer

`transferNft` chains build and send.

```ts
import { transferNft } from '@ton/appkit';

const response = await transferNft(appKit, {
  nftAddress: 'EQ-NFT-CONTRACT',
  recipientAddress: 'EQ-NEW-OWNER',
});
```

## Track settlement

NFT transfers settle through the same trace API as other transfers. Treat the wallet response as "user signed" and confirm with `getTransactionStatus`.

```ts
import { getTransactionStatus } from '@ton/appkit';

const status = await getTransactionStatus(appKit, { boc: response.boc });
if (status.status === 'completed') {
  // Refetch the owner record.
}
```

## React hooks and components

| Hook            | Wraps                |
| --------------- | -------------------- |
| `useNfts`       | `getNfts`            |
| `useNftsByAddress` | `getNftsByAddress` |
| `useNft`        | `getNft`             |
| `useTransferNft`| `transferNft`        |

The React package also ships an `<NftItem>` component for rendering a single NFT card (image, name, collection badge, on-sale indicator).

```tsx
import { useNfts, useTransferNft, NftItem } from '@ton/appkit-react';

function MyGallery() {
  const { data } = useNfts();
  return <div>{data?.nfts.map((nft) => <NftItem key={nft.address} nft={nft} />)}</div>;
}

function GiftButton({ nftAddress, to }: { nftAddress: string; to: string }) {
  const transfer = useTransferNft();
  return (
    <button onClick={() => transfer.mutate({ nftAddress, recipientAddress: to })}>
      Gift NFT
    </button>
  );
}
```

## Failure modes

| Class             | Example                                          | Recovery                                                     |
| ----------------- | ------------------------------------------------ | ------------------------------------------------------------ |
| Stale ownership   | The NFT was sold between read and transfer.      | Read `owner` again before opening the wallet; re-render.     |
| Missing metadata  | Off-chain metadata host is down.                 | Render a fallback card; do not block the transfer.           |
| Insufficient gas  | A complex collection contract needs more gas.    | Pass `amount` explicitly to override the default allowance.  |
| Provider failure  | API client cannot reach the NFT contract.        | Retry with backoff; surface a connectivity hint.             |
| Recipient typo    | Caller passes a non-friendly address.            | Validate before opening the wallet.                          |

## Rules

- Verify the current `owner` before allowing a user-initiated transfer. NFTs change hands fast.
- Treat metadata as untrusted. Sanitize names and images and keep on-chain addresses authoritative for routing.
- Use `getTransactionStatus` to confirm settlement before updating the gallery in app state. The wallet response only proves the user signed.
- Do not concatenate NFT addresses for analytics keys. Use the contract address verbatim and the index as a numeric.

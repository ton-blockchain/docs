A **wallet** is the user's signing application. A **connector** is the transport AppKit uses to talk to that wallet. Connectors connect, disconnect, and report which wallets are paired. AppKit collects wallets across all connectors into a single connected list and tracks which one is selected.

## Wallet state

| Term             | Meaning                                                                                          |
| ---------------- | ------------------------------------------------------------------------------------------------ |
| Connected wallet | A wallet for which a connector has an active session. Multiple connected wallets are possible.   |
| Selected wallet  | The wallet that mutating actions and signing flows use. AppKit auto-selects when a list changes. |
| Connector        | A transport with an `id`, `type`, and methods to connect, disconnect, and report wallets.        |
| Capability       | A request type a wallet supports (transfer, sign data). Not every wallet supports every flow.    |

## Connectors

A connector implements the `Connector` interface:

```ts
interface Connector {
  readonly id: string;
  readonly type: string;
  readonly metadata: ConnectorMetadata;
  destroy(): void;
  connectWallet(network?: Network): Promise<void>;
  disconnectWallet(): Promise<void>;
  getConnectedWallets(): WalletInterface[];
}
```

Register connectors at construction or later:

```ts
import { AppKit } from '@ton/appkit';
import { createTonConnectConnector } from '@ton/appkit/tonconnect';

const appKit = new AppKit({
  connectors: [
    createTonConnectConnector({
      tonConnectOptions: { manifestUrl: 'https://example.com/tonconnect-manifest.json' },
    }),
  ],
});
```

Connector ids are unique. Adding a connector with an existing id replaces and destroys the old one.

## TonConnect

`createTonConnectConnector` is the built-in TonConnect connector. It wraps `@tonconnect/ui`.

| Option              | Meaning                                                                            |
| ------------------- | ---------------------------------------------------------------------------------- |
| `id`                | Connector id. Defaults to `'tonconnect'`.                                          |
| `metadata`          | Optional `name` and `iconUrl` for UI lists.                                        |
| `tonConnectOptions` | Options passed to the underlying `TonConnectUI`. `manifestUrl` is required there.  |
| `tonConnectUI`      | A pre-constructed `TonConnectUI` instance, when you need to share it across views. |

The TonConnect connector restores prior sessions on first read and resyncs to AppKit's default network whenever it changes.

## Lifecycle

```text
addConnector → connect → connector:connected → wallets:updated → wallets:selection-changed
```

| Step    | Behavior                                                                                                |
| ------- | ------------------------------------------------------------------------------------------------------- |
| Connect | `connect({ connectorId })` calls `connector.connectWallet(defaultNetwork)`.                             |
| Pair    | The connector emits `connector:connected` with the new wallet list and connector id.                    |
| Update  | AppKit aggregates wallets from every connector into one list and emits `wallets:updated`.               |
| Select  | If the previously selected wallet is gone, AppKit auto-selects the first wallet and emits the change.   |
| Restore | TonConnect-style connectors call `restoreConnection()` on first read; the same events fire.             |
| Drop    | `disconnect({ connectorId })` calls `disconnectWallet()`. The wallet list shrinks; selection follows.   |

## Actions

| Action                 | Shape    | Purpose                                                       |
| ---------------------- | -------- | ------------------------------------------------------------- |
| `connect`              | Mutation | Open the connector flow for a given `connectorId`.            |
| `disconnect`           | Mutation | End the connector session.                                    |
| `getConnectors`        | Query    | List every registered connector.                              |
| `getConnectorById`     | Query    | Look up a connector by id.                                    |
| `watchConnectors`      | Watcher  | Subscribe to the connector list.                              |
| `watchConnectorById`   | Watcher  | Subscribe to a single connector by id.                        |
| `getConnectedWallets`  | Query    | Read the aggregated list of connected wallets.                |
| `getSelectedWallet`    | Query    | Read the currently selected wallet, or `null`.                |
| `setSelectedWalletId`  | Mutation | Set the selected wallet by id (or `null` to clear).           |
| `watchConnectedWallets`| Watcher  | Subscribe to the connected wallet list.                       |
| `watchSelectedWallet`  | Watcher  | Subscribe to the selected wallet.                             |

## React hooks

| Hook                  | Wraps                                       |
| --------------------- | ------------------------------------------- |
| `useConnect`          | `connect`                                   |
| `useDisconnect`       | `disconnect`                                |
| `useConnectors`       | `getConnectors` / `watchConnectors`         |
| `useConnectorById`    | `getConnectorById` / `watchConnectorById`   |
| `useConnectedWallets` | `getConnectedWallets` / `watchConnectedWallets` |
| `useSelectedWallet`   | `getSelectedWallet` / `watchSelectedWallet` / `setSelectedWalletId` |
| `useAddress`          | Convenience selector over `useSelectedWallet`. |

```tsx
import { useConnect, useConnectors, useSelectedWallet } from '@ton/appkit-react';

function ConnectButton() {
  const connectors = useConnectors();
  const connect = useConnect();
  const tonConnect = connectors.find((c) => c.id === 'tonconnect');
  if (!tonConnect) return null;
  return (
    <button onClick={() => connect.mutate({ connectorId: tonConnect.id })}>
      Connect wallet
    </button>
  );
}

function Account() {
  const [wallet] = useSelectedWallet();
  if (!wallet) return null;
  return <span>{wallet.getAddress().toString()}</span>;
}
```

## Wallet identity

Each wallet exposes the same minimum surface:

```ts
interface WalletInterface {
  readonly connectorId: string;
  getAddress(): UserFriendlyAddress;
  getPublicKey(): Hex;
  getNetwork(): Network;
  getWalletId(): string;
  sendTransaction(request: TransactionRequest): Promise<SendTransactionResponse>;
  signData(payload: SignDataRequest): Promise<SignDataResponse>;
}
```

Wallet ids are derived from `(network, address)`. The same address on a different network produces a different wallet object.

## Rules

- Do not call mutations or signing actions while `getSelectedWallet` is `null`. The action will throw.
- Do not assume every connector supports every flow. Catch rejections from the wallet and surface a recovery option.
- Do not pin a stale `selectedWalletId` in app state. The value can be reset by the connector at any moment (disconnect, network change, restore).
- Do not call connectors directly. Use the public actions or React hooks so AppKit's event flow stays consistent.

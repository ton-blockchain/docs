A **wallet** is the user's signing application. A **connector** is the transport AppKit uses to talk to that wallet. Connectors connect, disconnect, and report which wallets are paired. AppKit collects wallets across all connectors into a single connected list and tracks which one is **selected** — that is the wallet mutating actions and signing flows will use.

## Wallet state

| Term             | Meaning                                                                                          |
| ---------------- | ------------------------------------------------------------------------------------------------ |
| Connected wallet | A wallet for which a connector has an active session. Multiple connected wallets are possible.   |
| Selected wallet  | The wallet that mutating actions and signing flows use. AppKit auto-selects when the list changes. |
| Connector        | A transport with an `id`, `type`, and methods to connect, disconnect, and report wallets.        |
| Capability       | A request type a wallet supports (transfer, sign data). Not every wallet supports every flow.    |

## Connectors

A connector implements the `Connector` interface. Apps register connectors when constructing `AppKit`, or later through `addConnector`. Connector ids are unique — adding a connector with an existing id replaces and destroys the previous one.

```ts
interface Connector {
  readonly id: string;
  readonly type: string;
  readonly metadata: ConnectorMetadata;
  destroy(): void;
  connectWallet(network?: Network): Promise<void>;
  disconnectWallet(): Promise<void>;
  getConnectedWallets(): WalletInterface[];
}
```

The bundled `createTonConnectConnector` covers most apps. It wraps `@tonconnect/ui`, restores prior sessions on first read, and resyncs to AppKit's default network whenever it changes.

```ts
import { AppKit } from '@ton/appkit';
import { createTonConnectConnector } from '@ton/appkit/tonconnect';

const appKit = new AppKit({
  connectors: [
    createTonConnectConnector({
      tonConnectOptions: { manifestUrl: 'https://example.com/tonconnect-manifest.json' },
    }),
  ],
});
```

## Lifecycle

```text
addConnector → connect → connector:connected → wallets:updated → wallets:selection-changed
```

`connect({ connectorId })` calls into the connector to open its flow. When the user pairs a wallet, the connector emits `connector:connected` with the new wallet list. AppKit aggregates wallets from every connector into a single list and emits `wallets:updated`. If the previously selected wallet is gone, AppKit auto-selects the first wallet and emits `wallets:selection-changed`. `disconnect` runs the same chain in reverse.

## Wallet identity

Each wallet exposes the same minimum surface — address, public key, network, and the two operations (`sendTransaction`, `signData`) that flow through to its connector.

```ts
interface WalletInterface {
  readonly connectorId: string;
  getAddress(): UserFriendlyAddress;
  getPublicKey(): Hex;
  getNetwork(): Network;
  getWalletId(): string;
  sendTransaction(request: TransactionRequest): Promise<SendTransactionResponse>;
  signData(payload: SignDataRequest): Promise<SignDataResponse>;
}
```

Wallet ids are derived from `(network, address)`. The same address on a different network produces a different wallet object — apps that move users between mainnet and testnet should not assume id stability across that boundary.

## Rules

- Do not call mutations or signing actions while `getSelectedWallet` is `null`. The action will throw.
- Do not assume every connector supports every flow. Catch rejections from the wallet and surface a recovery option.
- Do not pin a stale `selectedWalletId` in app state. The value can be reset by the connector at any moment (disconnect, network change, restore).
- Do not call connectors directly. Use the public actions or React hooks so AppKit's event flow stays consistent.

UI widgets are end-to-end React components in `@ton/appkit-react` that render a complete flow on top of an AppKit manager — wallet connect, send, swap, staking, and onramp — with optional render-prop customisation.

## Widget set

| Widget                  | Flow                                       |
| ----------------------- | ------------------------------------------ |
| `TonConnectButton`      | Open the wallet connect dialog.            |
| `Send`                  | Send a prepared transaction request.       |
| `SendTonButton`         | Send Toncoin to an address.                |
| `SendJettonButton`      | Send a jetton to an address.               |
| `SwapWidget`            | Quote and execute a DEX swap.              |
| `StakingWidget`         | Quote and execute a stake or unstake.      |
| `OnrampWidget`          | Fiat-to-crypto onramp through a provider.  |
| `CryptoOnrampWidget`    | Crypto-to-crypto onramp through a bridge.  |
| `TonPayWidget`          | TON Pay-specific onramp UI.                |

Default styles are imported from `@ton/appkit-react/styles.css`.

## Provider, hooks, UI

Each widget is composed of three parts.

| Part                   | Role                                                            |
| ---------------------- | --------------------------------------------------------------- |
| `*WidgetProvider`      | Owns the widget's state machine and binds to the AppKit manager. |
| `use*Context`          | Reads widget state for custom rendering.                        |
| `*WidgetUI`            | Renders the default look.                                       |

Use these directly when a widget must be split across the React tree, for example a header amount input with a footer submit button.

## Render-prop customisation

Pass a child function to keep the widget's state and replace the layout. Each widget exposes typed render props (e.g. `SwapWidgetRenderProps`, `StakingWidgetRenderProps`).

```tsx
<SwapWidget tokens={tokens} network={Network.mainnet()}>
  {({ fromAmount, setFromAmount, toAmount, isQuoteLoading, sendSwapTransaction, canSubmit }) => (
    <CustomLayout ... />
  )}
</SwapWidget>
```

## Required context

Widgets must be rendered inside `AppKitProvider`. Most widgets require a `network` prop and the matching manager to be registered on the AppKit instance. A `SwapWidget` without a registered swap provider will not produce quotes.

## Boundaries

Widgets are vertical flows, not a design system. Custom layouts should use the render-prop form or the underlying hooks (`useSwapQuote`, `useStakingQuote`, `useOnrampQuote`, …), not theme overrides.

## Rule

Pick the widget level by how much control you need. Use the default UI for prototyping. Use the render-prop form for branded UI. Use hooks directly when the flow does not match a single widget.

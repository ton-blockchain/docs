Providers connect AppKit to external data and DeFi services. Four kinds matter: the **API client** that reads chain state, the **streaming provider** that pushes live updates, the **swap provider** that quotes and builds DEX transactions, and the **staking provider** that quotes and builds stake transactions. Each kind is owned by a matching manager on the `AppKit` instance.

| Type               | Use                                                                  | Manager                       |
| ------------------ | -------------------------------------------------------------------- | ----------------------------- |
| API client         | Reads blockchain data such as balances, jettons, and NFTs.           | `AppKit.networkManager`       |
| Streaming provider | Pushes live balance, transaction, and jetton updates over WebSocket. | `AppKit.streamingManager`     |
| Swap provider      | Returns swap quotes and builds swap transactions.                    | `AppKit.swapManager`          |
| Staking provider   | Returns staking quotes, balance, and builds stake transactions.      | `AppKit.stakingManager`       |

## How they are registered

There are two registration paths. **API clients** and **streaming providers** are bound to a network: configure them through the `networks` field of `AppKitConfig`, and AppKit constructs the matching streaming provider automatically. **Swap** and **staking** providers are generic: pass them to `providers` at construction or register them later with `registerProvider`. Each provider declares its `type` so AppKit can route it to the correct manager.

```ts
import { AppKit, Network, registerProvider } from '@ton/appkit';
import { createOmistonSwapProvider } from '@ton/appkit/swap/omniston';
import { createTonstakersProvider } from '@ton/appkit/staking/tonstakers';

const appKit = new AppKit({
  networks: {
    [Network.mainnet().chainId]: { url: 'https://toncenter.com/api/v2/jsonRPC' },
    [Network.testnet().chainId]: { url: 'https://testnet.toncenter.com/api/v2/jsonRPC' },
  },
  providers: [createOmistonSwapProvider()],
});

registerProvider(appKit, createTonstakersProvider());
```

AppKit ships with TonCenter and TonApi as the API clients and streaming providers, Omniston and DeDust as swap providers, and Tonstakers as a staking provider. Custom DeFi backends can be registered through the same `registerProvider` path by implementing the swap or staking interface.

## Two-step transactions

Swap and staking actions follow a quote-then-build pattern. `getSwapQuote` and `getStakingQuote` return external offers — fees, settlement amounts, routing data — and `buildSwapTransaction` / `buildStakeTransaction` turn an accepted quote into a `TransactionRequest`. Building the request does not broadcast it; the request still has to flow through `sendTransaction` and the connected wallet.

```ts
import { AppKit, getSwapQuote, buildSwapTransaction, sendTransaction } from '@ton/appkit';

const quote = await getSwapQuote(appKit, {
  amount: '1.0',
  from: 'EQ-TON-MASTER',
  to: 'EQ-USDT-MASTER',
  network,
});
const request = await buildSwapTransaction(appKit, {
  quote,
  userAddress: wallet.getAddress().toString(),
});
await sendTransaction(appKit, request);
```

This separation matters because quotes go stale. The settlement amount the user saw when the quote was fetched is not guaranteed when the transaction lands on chain — refetch the quote before building, and check the response again before crediting product state.

## Custom providers

Apps that integrate their own DEX or staking backend implement a small interface. The shape is the same as the bundled providers and gets registered through `registerProvider`. Provider ids are singletons; registering a second provider with the same `providerId` replaces the previous one.

## Rules

- Provider responses are external data. Validate critical outcomes (settlement amounts, addresses, signatures) before changing product state.
- Quotes go stale. Build the transaction soon after fetching the quote, or refetch before showing the user a final price.
- Do not assume a provider is registered. Call `getStakingProviders` or `hasStreamingProvider` and degrade gracefully when the list is empty.
- Treat provider ids as singletons. Registering a second provider with the same id replaces the previous one.

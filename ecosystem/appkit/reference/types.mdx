Types describe public request, result, and configuration objects.

## Core types

| Type                 | Description                       |
| -------------------- | --------------------------------- |
| `AppKitConfig`       | AppKit runtime configuration.     |
| `Network`            | Chain and API client description. |
| `Connector`          | Wallet connection transport.      |
| `Wallet`             | Connected wallet state.           |
| `TransactionRequest` | Wallet transaction request.       |
| `SignDataRequest`    | Wallet signing request.           |
| `Jetton`             | Jetton metadata and identity.     |
| `NFT`                | NFT metadata and identity.        |
| `Swap`               | Swap quote and transaction data.  |
| `Staking`            | Staking provider and quote data.  |

## Object format

### AppKitConfig

This object configures networks, connectors, providers, and default AppKit behavior.

| Field        | Type                            | Description                                            |
| ------------ | ------------------------------- | ------------------------------------------------------ |
| `networks`   | `Record<string, NetworkConfig>` | Networks available to the app.                         |
| `connectors` | `Connector[]`                   | Optional. Wallet connectors available to users.        |
| `providers`  | `Provider[]`                    | Optional. Data, swap, staking, or streaming providers. |

See [Schema](/ecosystem/appkit/schema) for canonical object pages.

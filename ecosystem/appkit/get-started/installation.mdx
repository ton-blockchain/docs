Before installing AppKit, add a small browser polyfill that its dependencies rely on. The AppKit packages themselves, along with the `AppKit` instance, connectors, and providers, are covered on the following pages.

## Buffer polyfill

AppKit relies on `@ton/core` and `@ton/crypto`, and both libraries use the `Buffer` global from Node.js. Browsers do not provide it, and modern bundlers (Vite, esbuild, Rollup, Parcel) do not add it automatically. Without the polyfill the first call into `@ton/core` — usually address parsing — throws `ReferenceError: Buffer is not defined` before the application has a chance to run.

There are several ways to polyfill `Buffer` — through bundler plugins, framework-level providers, or by hand. The approach below is one option; choose whichever fits the project's tooling best.

The polyfill needs to load before any AppKit import. The reliable way to ensure that is to load it from a separate module in the HTML entry, ahead of the application script — relying on import order inside the entry file is more fragile.

The example below uses Vite. For other module-based bundlers (Webpack, Parcel, Rollup) the steps are the same: a separate polyfill module loaded from the HTML entry before the application script.

<Steps>
  <Step title="Install the polyfill package">
    ```shell
    npm i buffer
    ```
  </Step>

  <Step title="Create a polyfill module">
    Add a file that imports the `buffer` package and assigns `Buffer` to `window`. Use `.ts` in TypeScript projects, `.js` otherwise.

    ```ts
    // src/buffer.ts
    import { Buffer } from 'buffer';

    window.Buffer = Buffer;
    ```
  </Step>

  <Step title="Load it before the application entry">
    In `index.html`, add a `<script type="module">` for the polyfill above the one that loads the application. Bundlers evaluate module scripts in document order, so the polyfill finishes before any application import is parsed.

    ```html
    <!-- index.html -->
    <body>
      <div id="root"></div>

      <script type="module" src="/src/buffer.ts"></script>
      <script type="module" src="/src/main.tsx"></script>
    </body>
    ```
  </Step>
</Steps>

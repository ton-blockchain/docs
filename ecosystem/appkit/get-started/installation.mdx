Install AppKit packages into your project. This page covers package selection and bundler setup. Code samples that use AppKit live on later pages.

## Pick a path

| Path                                                                  | Use when                                                              |
| --------------------------------------------------------------------- | --------------------------------------------------------------------- |
| [React App](/ecosystem/appkit/get-started/installation/react-app)     | Your app uses React and you want hooks, components, and styles.       |
| [Vanilla JS](/ecosystem/appkit/get-started/installation/vanilla-js)   | Your app does not use React and you call AppKit core actions.         |

Both paths share the core `@ton/appkit` package. The React path adds `@ton/appkit-react` for the provider, hooks, and styles.

## Browser polyfill

AppKit itself does not import `Buffer` directly, but several of its dependencies expect a `Buffer` global in the browser. Pick one of the two approaches below.

### Option 1: Manual polyfill (recommended)

Create a `polyfills.ts` file and import it before anything else in your entry file. This keeps the bundler config untouched.

```ts
// src/polyfills.ts
import { Buffer } from 'buffer';

declare global {
  interface Window {
    global: typeof globalThis;
    Buffer: typeof Buffer;
  }
}

window.global = window;
window.Buffer = Buffer;
```

```tsx
// src/main.tsx — first import, before React or AppKit
import './polyfills';

import { StrictMode } from 'react';
import { createRoot } from 'react-dom/client';
import { App } from './app';

createRoot(document.getElementById('root')!).render(
  <StrictMode>
    <App />
  </StrictMode>,
);
```

### Option 2: Vite alias

If you cannot run code before module evaluation, alias the polyfill at the bundler level instead.

```ts
// vite.config.ts
import { defineConfig } from 'vite';

export default defineConfig({
  resolve: {
    alias: { buffer: 'buffer/' },
  },
});
```

For webpack and other bundlers, configure the equivalent alias and the `process` and `Buffer` globals.

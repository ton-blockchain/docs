This page answers common AppKit integration questions. Each answer is short and links to the canonical page for details.

## What is AppKit?

AppKit is a TypeScript toolkit for connecting TON wallets, building transaction requests, reading wallet data, and using React hooks and components.

## Is AppKit a wallet?

No. AppKit does not hold private keys and does not sign transactions. Wallets sign transactions and data. AppKit prepares requests and sends them through connectors.

## Does AppKit store private keys?

No. Private keys stay in the user's wallet. AppKit may store connector session data so the application can restore a connection.

## Which wallets are supported?

Wallet support depends on the configured connectors. TonConnect connectors use the TON Connect wallet ecosystem. See [Wallets and connectors](/ecosystem/appkit/concepts/wallets-and-connectors).

## Can I use AppKit without React?

Yes. Use `@ton/appkit` actions directly. React support is provided by `@ton/appkit-react` and is optional.

## Can I use AppKit on the server?

Use AppKit in the browser for wallet interaction. Use your server for order state, signature verification, transaction verification, and private API keys.

## How do I switch between mainnet and testnet?

Configure networks in the AppKit instance and set the default network for the current runtime. See [Networks](/ecosystem/appkit/concepts/networks).

## How do I know a transaction succeeded?

A wallet approval means the wallet accepted the request. It does not prove final settlement. Track the transaction and verify the result with an indexer or backend. See [Track transactions](/ecosystem/appkit/build/track-transactions).

## Why does a wallet reject a transaction?

Common reasons include user rejection, expired request, unsupported payload, wrong network, insufficient balance, or wallet capability mismatch.

## How do I verify signed data?

Request a signature in the browser and verify it on the backend against the expected address, domain, payload, timestamp, and nonce. See [Sign data](/ecosystem/appkit/build/sign-data).

## How do I handle jetton decimals?

Read jetton metadata and convert human input to the token's base units before building a transaction. Never assume all jettons use the same decimals.

## What data must be verified on the backend?

Verify paid amount, destination, asset, user identity if needed, transaction inclusion, and order state. The browser is not the source of truth for settlement.

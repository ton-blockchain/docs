Use this section to learn the AppKit surface one piece at a time. Each page introduces one part of the toolkit with the smallest example that explains it.

AppKit ships in three layers. `@ton/appkit` is the core: it owns the `AppKit` instance, the connectors, the actions, and the type definitions, and it works on its own from plain TypeScript. `@ton/appkit-react` is a thin layer on top that exposes the same surface as React hooks and ready-to-mount components, plus a default stylesheet. The optional DeFi and streaming providers (`@ton/appkit/swap/*`, `@ton/appkit/staking/*`, and the streaming providers registered on `appKit.streamingManager`) plug AppKit into external services and live data.

Read Get started for orientation. Each page here defines one piece of the surface, shows the smallest example that explains it, and stops. When you need to actually build a flow — connect, read, send, sign — switch to the [How to](/ecosystem/appkit/howto) section, which extends the same code patterns into full task recipes.

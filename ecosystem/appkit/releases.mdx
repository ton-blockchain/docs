AppKit release notes should be read before upgrading package versions.

## Release format

Each release entry records:

- date and package version;
- added APIs;
- changed behavior;
- deprecated APIs;
- migration notes;
- compatibility notes for React, providers, and package entrypoints.

## Current development line

AppKit is documented from the public package exports in `@ton/appkit` and `@ton/appkit-react`.

| Area      | Upgrade check                                                                |
| --------- | ---------------------------------------------------------------------------- |
| Core      | Rebuild the `AppKit` instance configuration and run wallet connection tests. |
| Actions   | Check parameter and return type changes.                                     |
| React     | Check provider, hook, and component changes.                                 |
| Providers | Check swap and staking provider compatibility.                               |
| Schema    | Check request object and config object changes.                              |

## Upgrade checklist

1. Update AppKit packages together.
1. Run type checking.
1. Test wallet connection on testnet.
1. Test transaction approval and rejection.
1. Verify transaction tracking against your backend.
1. Review deprecated APIs in [Reference](/ecosystem/appkit/reference).

## Deprecations

Document deprecated APIs here before removing them. Each deprecation should include the replacement API and the first release where the old API stops working.

Use this checklist before sending mainnet users through AppKit flows.

## Manifest and identity

- Host a stable TON Connect manifest.
- Use the production application name and icon.
- Keep the manifest URL available over HTTPS.

## Networks

- Use mainnet only when the flow is ready for real funds.
- Block actions when a wallet or configured provider is on the wrong network.
- Test all flows on testnet first.

## Providers

- Use provider API keys according to provider rules.
- Keep private keys and privileged API keys on the server.
- Add fallback behavior when provider data is delayed.

## Transactions

- Set clear recipients, amounts, payloads, and expiration times.
- Display what the wallet will ask the user to approve.
- Treat wallet approval and on-chain settlement as separate states.
- Verify paid orders on the backend.

## Errors

- Handle user rejection as a normal path.
- Log request IDs, wallet address, network, asset, amount, and transaction state.
- Never expose secrets in frontend logs.

## Security notes

AppKit prepares requests. It does not prove that an order was paid or that signed data is valid. Verify critical state outside the browser.

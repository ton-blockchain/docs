`AppKitConfig` defines networks, connectors, and providers for one application runtime.

## Basic shape

```ts
import { AppKit, Network, createTonConnectConnector } from '@ton/appkit';

export const appKit = new AppKit({
  networks: {
    [Network.mainnet().chainId]: {
      apiClient: { url: 'https://toncenter.com', key: import.meta.env.VITE_TONCENTER_API_KEY },
    },
  },
  connectors: [
    createTonConnectConnector({
      tonConnectOptions: { manifestUrl: 'https://example.com/tonconnect-manifest.json' },
    }),
  ],
});
```

## Fields

| Field        | Type                            | Description                                             |
| ------------ | ------------------------------- | ------------------------------------------------------- |
| `networks`   | `Record<string, NetworkConfig>` | Networks available to the application.                  |
| `connectors` | `Connector[]`                   | Optional. Wallet connectors that users can choose from. |
| `providers`  | `Provider[]`                    | Optional. Swap, staking, streaming, or data providers.  |

## Rules

- Create one AppKit instance for one app runtime.
- Keep private API keys on the server when the provider supports server-side access.
- Use testnet configuration for development transaction tests.
- Keep connector setup close to application identity and manifest configuration.

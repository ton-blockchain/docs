Install the core package, React bindings, and peer packages used by the default React setup.

## Package install

```bash
npm i @ton/appkit @ton/appkit-react @tanstack/react-query @tonconnect/ui-react @ton/core buffer
```

Use the package manager used by your application. Install AppKit packages together so core and React exports stay compatible.

## Public entrypoints

| Entrypoint                       | Use                                                       |
| -------------------------------- | --------------------------------------------------------- |
| `@ton/appkit`                    | Core instance, actions, connectors, providers, and types. |
| `@ton/appkit/queries`            | Query options, mutation options, and query keys.          |
| `@ton/appkit-react`              | React provider, hooks, and UI components.                 |
| `@ton/appkit-react/styles.css`   | Default styles for React components.                      |
| `@ton/appkit/swap/omniston`      | Omniston swap provider.                                   |
| `@ton/appkit/swap/dedust`        | DeDust swap provider.                                     |
| `@ton/appkit/staking/tonstakers` | Tonstakers staking provider.                              |

## Browser polyfill

Some bundlers require a `buffer` alias when TON packages use Node-compatible primitives.

```ts
import { defineConfig } from 'vite';

export default defineConfig({
  resolve: {
    alias: {
      buffer: 'buffer/',
    },
  },
});
```

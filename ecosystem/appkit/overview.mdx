## What is AppKit

AppKit gives one place to configure networks, connectors, providers, and React integration around TON wallet operations. It prepares requests, tracks application state, and bridges the application interface to wallets and providers.

Wallets still sign transactions and data. Backends still verify orders, signatures, and business rules.

Reach for AppKit when an application needs to:

- connect a TON wallet
- request Toncoin, jetton, NFT, swap, staking, or signing operations
- read wallet data and asset balances

## Packages

| Package                          | Use                                                              |
| -------------------------------- | ---------------------------------------------------------------- |
| `@ton/appkit`                    | Core AppKit instance, actions, connectors, providers, and types. |
| `@ton/appkit/queries`            | Query and mutation helpers for data fetching.                    |
| `@ton/appkit-react`              | React provider, hooks, components, and core re-exports.          |
| `@ton/appkit-react/styles.css`   | Default component styles.                                        |
| `@ton/appkit/swap/omniston`      | Omniston swap provider.                                          |
| `@ton/appkit/swap/dedust`        | DeDust swap provider.                                            |
| `@ton/appkit/staking/tonstakers` | Tonstakers staking provider.                                     |

## Examples

The [`appkit-minter`](https://github.com/ton-connect/kit/tree/main/apps/appkit-minter) example app wires AppKit into a Vite + React app: wallet connect, balance and jetton reads, transaction sending, and streaming watches in one place.

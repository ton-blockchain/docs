## What TON Connect is

**TON Connect** is the standard wallet connection protocol for the TON blockchain.

It links a dApp to a user's wallet over an end-to-end encrypted session so the app can read the connected address, request signatures, and send transactions — without ever touching the user's keys.

TON Connect allows connecting TON wallets to the frontend of web applications or [Telegram Mini Apps](https://core.telegram.org/bots/webapps).

<Aside type="note">
TON Connect does not provide integration with the TON blockchain. You need to implement it manually or use [AppKit](/ecosystem/appkit/overivew) — an SDK for integrating TON into dApps.
</Aside>

### Build a wallet

TON Connect does not provide an SDK for wallets. If you're a wallet developer, take one of these approaches:

- Implement the protocol directly based on the [TON Connect specification](https://github.com/the-ton-tech/ton-connect/tree/feature/refine-protocol). Additionally, you can use [`@tonconnect/protocol`](/ecosystem/ton-connect/api-reference/protocol) for interfaces.
- Use [WalletKit](/ecosystem/walletkit/overview) — an SDK for wallet services with built-in support for TON Connect

## API reference

The dApp-facing packages live in the [`ton-connect/sdk`](https://github.com/ton-connect/sdk) monorepo:

- [`@tonconnect/ui-react`](/ecosystem/ton-connect/api-reference/ui-react) — hooks and prebuilt components for React.
- [`@tonconnect/ui`](/ecosystem/ton-connect/api-reference/ui) — framework-agnostic UI, same components without React bindings.
- [`@tonconnect/sdk`](/ecosystem/ton-connect/api-reference/sdk) — headless connector for server-side flows or custom UI.
- [`@tonconnect/protocol`](/ecosystem/ton-connect/api-reference/protocol) — wire-format types and session cryptography for wallet/SDK implementers.

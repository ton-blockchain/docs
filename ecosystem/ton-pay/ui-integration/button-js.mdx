`@ton-pay/ui` is the framework-agnostic hosted widget SDK for plain HTML/JS pages or non-React integrations. It renders a TON Pay button when needed and opens the checkout UI in a fullscreen iframe appended to `document.body`.

## Install

Choose one of the supported delivery methods:

```bash
npm install @ton-pay/ui
```

Or load the browser bundle directly:

```html
<script src="https://unpkg.com/@ton-pay/ui/dist/tonpay-sdk.js"></script>
```

<Aside type="note">
  The CDN bundle exposes `window.TonPay`, while bundler users import the same API from `@ton-pay/ui`. Calls such as `TonPay.render(...)` in the examples below map to the named ESM export `render(...)`, and the same applies to `createWidget` and other APIs.
</Aside>

## Option 1: render the built-in button from a script tag

```html
<div id="tonpay-checkout"></div>

<script src="https://unpkg.com/@ton-pay/ui/dist/tonpay-sdk.js"></script>
<script>
  const widget = window.TonPay.render("#tonpay-checkout", {
    amount: 12.34,
    chain: "testnet",
    comment: "Order #123",
    currency: "TON",
    itemTitle: "Electric Guitar",
    recipientWalletAddress: "<RECIPIENT_WALLET_ADDRESS>",
    onError(payload) {
      console.error(payload.code, payload.message);
    },
    onSuccess(payload) {
      console.log("Payment completed", payload);
    }
  });
</script>
```

## Option 2: use the ESM API with a bundler

```ts
import { render } from "@ton-pay/ui";

const widget = render("#tonpay-checkout", {
  amount: 25,
  chain: "mainnet",
  comment: "Order #8451",
  currency: "USDT",
  itemTitle: "Premium plan",
  recipientWalletAddress: "<RECIPIENT_WALLET_ADDRESS>",
  onReady() {
    console.log("Widget ready");
  }
});
```

## Option 3: headless mode with a custom trigger

Use `createWidget()` when the host application renders its own button or CTA.

```html
<button id="custom-pay-button">Pay now</button>

<script src="https://unpkg.com/@ton-pay/ui/dist/tonpay-sdk.js"></script>
<script>
  const widget = window.TonPay.createWidget({
    amount: 8,
    chain: "testnet",
    currency: "TON",
    recipientWalletAddress: "<RECIPIENT_WALLET_ADDRESS>"
  });

  document
    .getElementById("custom-pay-button")
    .addEventListener("click", () => widget.open());
</script>
```

## API

### `TonPay.render(target, options)`

Create a widget instance and mount the built-in button immediately.

```js
const widget = TonPay.render("#tonpay-checkout", {
  amount: 15,
  recipientWalletAddress: "<RECIPIENT_WALLET_ADDRESS>"
});
```

### `TonPay.createWidget(options)`

Create a widget instance without mounting a button.

```js
const widget = TonPay.createWidget({
  amount: 15,
  recipientWalletAddress: "<RECIPIENT_WALLET_ADDRESS>"
});
```

### Instance methods

- `mount(target)` renders the built-in TON Pay button.
- `unmount()` removes the built-in button but keeps the widget instance alive.
- `open()` opens the fullscreen hosted widget.
- `close()` closes the hosted widget.
- `update(partialOptions)` updates checkout or button options.
- `destroy()` removes the button, iframe, and event listeners.

## Options

### Checkout options

| Option                   | Type                     | Default             | Description                                                                                                         |
| ------------------------ | ------------------------ | ------------------- | ------------------------------------------------------------------------------------------------------------------- |
| `amount`                 | `number \| string`       | required            | Payment amount in asset units.                                                                                      |
| `recipientWalletAddress` | `string`                 | required            | Recipient wallet address.                                                                                           |
| `chain`                  | `"mainnet" \| "testnet"` | `"mainnet"`         | TON Pay environment used for session bootstrap.                                                                     |
| `currency`               | `string`                 | -                   | Asset symbol shown to the user.                                                                                     |
| `asset`                  | `string`                 | `currency ?? "TON"` | Asset identifier, including jettons.                                                                                |
| `itemTitle`              | `string`                 | -                   | UI-only text displayed in the widget.                                                                               |
| `comment`                | `string`                 | -                   | Payment comment added to the blockchain transaction. It maps to `commentToRecipient`.                               |
| `referenceId`            | `string`                 | -                   | Merchant-defined identifier passed through the checkout and echoed back in `onSuccess` payloads for reconciliation. |

### Button options

| Option         | Type                           | Default     | Description                              |
| -------------- | ------------------------------ | ----------- | ---------------------------------------- |
| `variant`      | `"long" \| "short"`            | `"long"`    | Button label variant.                    |
| `text`         | `string`                       | -           | Override the default button label.       |
| `bgColor`      | `string`                       | `"#0098EA"` | Button background color.                 |
| `textColor`    | `string`                       | `"#FFFFFF"` | Button text color.                       |
| `borderRadius` | `number \| string`             | `8`         | Button border radius.                    |
| `width`        | `number \| string`             | `"100%"`    | Button width.                            |
| `height`       | `number \| string`             | `44`        | Button height.                           |
| `fontFamily`   | `string`                       | `"inherit"` | Button font family.                      |
| `disabled`     | `boolean`                      | `false`     | Disable the button.                      |
| `isLoading`    | `boolean`                      | `false`     | Force a loading state from the host app. |
| `className`    | `string`                       | -           | Extra button class name.                 |
| `style`        | `Partial<CSSStyleDeclaration>` | -           | Extra inline styles for the button.      |

### Lifecycle callbacks

| Option          | Type                | Description                                             |
| --------------- | ------------------- | ------------------------------------------------------- |
| `onReady`       | `() => void`        | Called when the visible widget is ready.                |
| `onClose`       | `() => void`        | Called when the widget closes.                          |
| `onRouteChange` | `(payload) => void` | Called when the iframe route changes.                   |
| `onError`       | `(payload) => void` | Called for session bootstrap and runtime widget errors. |
| `onSuccess`     | `(payload) => void` | Called when the widget finishes successfully.           |

## Fullscreen behavior

The hosted widget always renders as a fullscreen overlay under `document.body`. This means:

- It is not clipped by the button container.
- It is not limited by the host page layout.
- It behaves the same on plain sites and [Telegram Mini Apps](/ecosystem/tma/overview).

## Best practices

- Use `chain="testnet"` during local integration.
- Wrap callbacks with logging or UI feedback for error handling.
- Persist callback payload identifiers for checkout tracking on the merchant side.
- Use headless mode when the host app needs a fully custom trigger.

## Troubleshooting

<Accordion
  title="If the button does not render"
>
  - Ensure the target selector exists before calling `render()` or `mount()`.
  - Check the browser console for script loading or runtime errors.
</Accordion>

<Accordion
  title="If the widget does not open"
>
  - Verify that the page can reach the TON Pay API origin for the selected `chain`.
  - Inspect `onError` payloads for bootstrap failures.
</Accordion>

<Accordion
  title="If the widget looks clipped"
>
  - The widget should not be constrained by host containers.
  - If clipping occurs, check whether custom host CSS overrides global iframe or `position: fixed` behavior.
</Accordion>

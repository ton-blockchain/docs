TON Pay SDK is a developer toolkit for integrating payment functionality on the TON blockchain. It supports web applications, [Telegram Mini Apps](/ecosystem/tma/overview), backend services, and bots. The SDK provides a unified interface for creating, verifying, and managing payments, abstracting blockchain-specific logic.

<Aside
  type="caution"
>
  TON Pay is not a payment processor.
</Aside>

## Features

- Payment creation: one-time transfers, recurring payments, and subscriptions, with
  customizable reference IDs and memo fields.
- Transaction tracking: transaction status tracking, confirmation handling, error reporting, and
  analytics.
- UI components: React hooks, ready-to-use payment components, and built-in
  status indicators.

## UI integration

While the SDK provides full programmatic control over payments, built-in UI components reduce integration time.

- UI components render a payment button and open the hosted TON Pay widget.
- The host application passes checkout parameters such as amount, recipient wallet address, asset, and comment.
- TON Pay handles session bootstrap, checkout routing, wallet connection, transaction signing, and result handling.

UI components provide a complete checkout experience for web applications. Select an integration based on the application stack:

- React – React component [`TonPayWidget`](/ecosystem/ton-pay/ui-integration/button-react).
- JavaScript – framework-agnostic SDK [`@ton-pay/ui`](/ecosystem/ton-pay/ui-integration/button-js).

## How TON Pay works

1. TON Pay backend generates a transaction message.
1. The wallet presents the transaction for review and requires a signature.
1. After the payment is submitted for signing, the SDK tracks its status using `reference` or `bodyBase64Hash`.
1. The transaction is executed and confirmed on the TON blockchain.

## Security

- Non-custodial: funds always remain under user control.
- Standard protocols: built on proven TON standards.
- Transparency: all transactions are verifiable on-chain.

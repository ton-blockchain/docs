`TonPayWidget` is the React component for TON Pay [`v0.4`](https://www.npmjs.com/package/@ton-pay/api?activeTab=versions). The hosted widget manages checkout state internally and returns lifecycle callbacks to the host app.

## Recommended integration

```tsx
import { TonPayWidget } from "@ton-pay/ui-react";

export function Checkout() {
  return (
    <TonPayWidget
      amount={3.5}
      chain="testnet"
      comment="Order #8451"
      currency="TON"
      itemTitle="Premium plan"
      recipientWalletAddress="<RECIPIENT_WALLET_ADDRESS>"
      onSuccess={(payload) => {
        console.log("Payment completed", payload);
      }}
      onError={(payload) => {
        console.error("Payment failed", payload);
      }}
    />
  );
}
```

## Widget capabilities

- Preload the widget session automatically.
- Open the iframe checkout on click.
- Manage wallet connection inside the widget.
- Handle checkout routing and result states.
- Handles payment status on the TON Pay side.
- Forwards success and error payloads to the host app.

## Success handling

```tsx
<TonPayWidget
  amount={12}
  chain="testnet"
  recipientWalletAddress="<RECIPIENT_WALLET_ADDRESS>"
  onSuccess={(payload) => {
    console.log("action:", payload.action);
    console.log("reference:", payload.reference);
    console.log("txHash:", payload.txHash);
  }}
/>
```

For direct wallet payments, `reference` is preserved in the success payload when available.

## Error handling

```tsx
<TonPayWidget
  amount={12}
  recipientWalletAddress="<RECIPIENT_WALLET_ADDRESS>"
  onError={(payload) => {
    console.error(payload.code, payload.message);
  }}
/>
```

Session bootstrap failures are normalized to `WidgetErrorPayload` and returned through `onError`.

- `comment` –  payment comment added to the blockchain transaction.
- `itemTitle` – UI-only text displayed in the widget.

## When to use server-side integration

Use [`@ton-pay/api`](/ecosystem/ton-pay/payment-integration/payments-server-side) only when the application requires flows outside the default widget checkout, such as:

- backend transfer creation;
- explicit transfer status lookups by `reference` or `bodyBase64Hash`.

## Migration from version 0.3.2 or lower

The current React integration is based on `TonPayWidget`, not the legacy `TonPayButton` + `useTonPay` integration.

Use the [migration guide](/ecosystem/ton-pay/payment-integration/migration) for the upgrade checklist and before-and-after examples.

## Create the first widget

<Steps>
  <Step
    title="Install the UI package"
  >
    Install `@ton-pay/ui-react`.

    ```bash
    npm install @ton-pay/ui-react
    ```

    <Aside
      type="tip"
    >
      `TonPayWidget` does not require `@ton-pay/api`. Install it separately only for server-side helpers or non-UI flows.
    </Aside>
  </Step>

  <Step
    title="Add the widget component"
  >
    Add `TonPayWidget` to the page and pass the required props.

    ```tsx
    import { TonPayWidget } from "@ton-pay/ui-react";

    <TonPayWidget
      // By default, the widget uses TON on mainnet.
      amount={12.34}
      recipientWalletAddress="<RECIPIENT_WALLET_ADDRESS>"
    />
    ```

    <Aside
      type="caution"
      title="Use testnet during development"
    >
      The snippet above defaults to `chain="mainnet"`. During development, set `chain="testnet"` until the wallet, asset, and checkout flow are ready for production.
    </Aside>

    Required props:

    - `amount` – payment amount in asset units.
    - `recipientWalletAddress` – recipient wallet address.

    Main checkout props:

    - `asset` – asset identifier. Use `TON`, `USDT`, or a jetton master address.
    - `chain` – TON Pay environment. Defaults to `mainnet`.
    - `itemTitle` – UI label displayed in the widget.
    - `comment` – payment comment added to the blockchain transaction.

    The full list of props is available in the [API reference](/ecosystem/ton-pay/api-reference).

    - `TonPayWidget` renders the payment button and opens the hosted TON Pay widget.
    - The host app passes checkout parameters.
    - TON Pay handles session bootstrap, checkout routing, wallet connection, transaction signing, and result handling.
  </Step>

  <Step
    title="Run the app"
  >
    Start the React app:

    ```bash
    npm run dev
    ```

    Open the local app URL and click the button to launch the widget.
  </Step>
</Steps>

## Full example

This example scaffolds a React app with Vite, installs `@ton-pay/ui-react`, and renders the `TonPayWidget` with common checkout props and result callbacks.

Replace `<RECIPIENT_WALLET_ADDRESS>` with a recipient wallet address.

```bash
npm create vite@latest tonpay-widget-demo -- --template react-ts
cd tonpay-widget-demo
npm install
npm install @ton-pay/ui-react
```

```tsx
// src/App.tsx
import { TonPayWidget } from "@ton-pay/ui-react";

export default function App() {
  return (
    <main>
      <section>
        <h1>TON Pay checkout</h1>

        <TonPayWidget
          amount={12.34}
          chain="testnet"
          asset="TON"
          comment="Order #123"
          itemTitle="Electric Guitar"
          recipientWalletAddress="<RECIPIENT_WALLET_ADDRESS>"
          onError={(payload) => {
            console.error(payload.code, payload.message);
          }}
          onSuccess={(payload) => {
            console.log("Payment completed", payload);
          }}
        />
      </section>
    </main>
  );
}
```

Common optional props:

- `chain` – TON Pay environment. Defaults to `mainnet`.
- `comment` –  payment comment added to the blockchain transaction.
- `itemTitle` – UI-only text displayed in the widget.
- `onSuccess` – callback invoked when the widget completes successfully.
- `onError` – callback invoked on widget runtime or session bootstrap errors.

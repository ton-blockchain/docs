Use this section to learn the AppKit surface one piece at a time. Each page introduces one part of the toolkit with the smallest example that explains it.

AppKit has three layers: `@ton/appkit` for the core instance, connectors, actions, and types; `@ton/appkit-react` for React hooks, components, and styles; and optional providers for DeFi and streaming integrations.

Each page here defines one piece of the surface, shows the smallest example that explains it, and stops. To build a connect, read, send, or sign flow, switch to the [How to](/ecosystem/appkit/howto) section, which extends the same code patterns into full task recipes.

## What you will learn

| Page                                                                       | What it covers                                                              |
| -------------------------------------------------------------------------- | --------------------------------------------------------------------------- |
| [Installation](/ecosystem/appkit/get-started/installation)                 | Add AppKit packages to a React or vanilla JavaScript project.               |
| [Connectors](/ecosystem/appkit/get-started/connectors)                     | Create the AppKit instance and connect wallets with `<TonConnectButton />`. |
| [Providers](/ecosystem/appkit/get-started/providers)                       | Configure API clients and plug in streaming, swap, and staking services.    |
| [Basic getter hooks](/ecosystem/appkit/get-started/basic-getter-hooks)     | Read wallet, address, balance, jetton, NFT, network, and transaction state. |
| [Streaming](/ecosystem/appkit/get-started/streaming)                       | Subscribe to live updates with `useWatchBalance`, `useWatchTransactions`.   |
| [Sending transactions](/ecosystem/appkit/get-started/sending-transactions) | Send Toncoin, jettons, NFTs, and prepared transactions with `<Send />`.     |

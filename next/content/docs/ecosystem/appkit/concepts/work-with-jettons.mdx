Jettons are TON's fungible-token standard. Each user holds tokens in a per-user **jetton wallet** contract derived from the **jetton master** address. AppKit exposes actions for reading balances, fetching metadata, resolving jetton wallet addresses, and building transfer requests.

Jetton balances and transfer amounts are strings in human-readable units. Conversion to the smallest unit uses the jetton's `decimals`, so read decimals from chain metadata before building a transfer. Do not assume that every jetton uses the same precision.

## Read jettons

`getJettons` returns the jettons held by the selected wallet. `getJettonInfo` reads metadata for a master contract, such as name, symbol, decimals, and image. `getJettonBalance` reads one jetton balance for an owner. `getJettonWalletAddress` derives the owner's per-user jetton wallet for a given master, which most apps do not need directly.

```ts
import { getJettons, getJettonInfo, getJettonBalance } from '@ton/appkit';

const owned = await getJettons(appKit);
// owned.jettons = [{ address, walletAddress, balance, decimalsNumber, info, isVerified, prices, ... }]
// owned.jettons[0].info = { name, symbol, image, description, ... }

const info = await getJettonInfo(appKit, { address: 'EQ-USDT-MASTER' });
// info.symbol === 'USDT', info.decimals === 6

const balance = await getJettonBalance(appKit, {
  jettonAddress: 'EQ-USDT-MASTER',
  ownerAddress: 'EQ-OWNER',
  jettonDecimals: 6,
});
```

`getJettons` returns `null` when no wallet is selected. `getJettonInfo` returns `null` when the jetton master is not found. `watchJettons` and `watchJettonsByAddress` subscribe to changes.

## Build and send a transfer

`createTransferJettonTransaction` resolves the selected wallet's jetton wallet contract, builds the internal transfer payload, and wraps it in a `TransactionRequest` with a default gas allowance. Pass the destination **owner address** as the recipient, not the recipient's jetton wallet address.

```ts
import { createTransferJettonTransaction, sendTransaction } from '@ton/appkit';

const request = await createTransferJettonTransaction(appKit, {
  jettonAddress: 'EQ-USDT-MASTER',
  recipientAddress: 'EQ-RECIPIENT',
  amount: '12.5',
  jettonDecimals: 6,
  comment: 'Invoice 42',
});
const response = await sendTransaction(appKit, request);
```

| Parameter            | Required | Meaning                                                                |
| -------------------- | -------- | ---------------------------------------------------------------------- |
| `jettonAddress`      | Yes      | Master contract address of the jetton.                                 |
| `recipientAddress`   | Yes      | Destination owner address (not their jetton wallet).                   |
| `amount`             | Yes      | Amount in jetton units, as a decimal string.                           |
| `jettonDecimals`     | Yes      | Decimal precision of the jetton. Read from `getJettonInfo` if unknown. |
| `comment`            | No       | Optional comment payload attached to the transfer.                     |

`createTransferJettonTransaction` is async because it queries the chain to derive the sender's jetton wallet address. It throws when no wallet is selected. The `transferJetton` shorthand builds and sends the transfer in one call.

## Track settlement

Jetton transfers settle through the same trace API as TON transfers. Call `getTransactionStatus` with the returned `boc`. A jetton transfer usually produces multiple internal messages; use `onchainMessages`, `pendingMessages`, and `totalMessages` to track progress. Treat only `status === 'completed'` as final.

```ts
import { getTransactionStatus } from '@ton/appkit';

const status = await getTransactionStatus(appKit, { boc: response.boc });
```

If the recipient has never held the jetton, the transfer may deploy the recipient's jetton wallet, which costs additional gas. The default allowance is designed to cover this case. If a token contract needs more gas, adjust the outer TON message amount on the built request, for example `request.messages[0].amount`.

## Rules

- Always pass the destination **owner address** as `recipientAddress`. AppKit derives the sender's jetton wallet contract and writes the destination owner address into the jetton transfer payload.
- Read decimals from chain metadata (`getJettonInfo`) before building a transfer. Different jettons can use different precision.
- Treat the `boc` from `sendTransaction` as "user signed". Verify settlement with `getTransactionStatus` before crediting an order.
- Refetch the user's jetton list and balances after settlement.
- Do not display jetton metadata without sanitizing — names and images are user-supplied.

Tolk has the following types:

- [numbers](/tolk/types/numbers) — `int`, `int32`, `uint64`, `coins`, and others.
- [boolean](/tolk/types/booleans) — `true` and `false`.
- [address](/tolk/types/address) — `internal`, `external`, and `none`.
- [cells](/tolk/types/cells) — containers with up to 1023 bits of data and up to 4 references to other cells, plus the cell manipulation primitives of TVM: [builders and slices](/tvm/builders-and-slices).
- [structures](/tolk/types/structures) — multiple fields grouped into one entity.
- [generics](/tolk/types/generics) — any struct can be generic `<T>`.
- [enums](/tolk/types/enums) — distinct types containing integer variants.
- [nullable types](/tolk/types/nullable) — `null` safety and safe casts.
- [union types](/tolk/types/unions) — variables holding one of several possible values.
- [strings](/tolk/types/strings) — string values stored as snake-encoded cells.
- [tensors](/tolk/types/tensors) — multiple values placed sequentially on the stack.
- [arrays](/tolk/types/tuples) — dynamically sized containers backed by TVM tuples.
- [maps](/tolk/types/maps) — key-value dictionaries.
- [callables](/tolk/types/callables) — first-class functions.
- [unknown](/tolk/types/unknown) — any value that is a single-slot TVM primitive.
- [`void` and `never`](/tolk/types/void-never) — both represent the absence of a value.

To make an interchangeable alias for an existing type, including structural types, there are [type aliases](/tolk/types/aliases).

Smart contracts run on a stack-based virtual machine, [TVM](/tvm/overview), which imposes specific rules on how values are represented at runtime. For example, [strings](/tolk/types/strings) are stored as snake-encoded cells, because TVM has no native string support.

All on-chain data and communication rely entirely on [cells](/tolk/types/cells), so the type system focuses on binary serialization and clear data relationships:

- [Type checks and casts](/tolk/types/type-checks-and-casts) covers casting with the unsafe `as` operator.
- [TVM stack representation](/tolk/types/overall-tvm-stack) summarizes how types map to the TVM stack.
- [Serialization](/tolk/types/overall-serialization) describes how types serialize and relate to [TL-B](/foundations/tlb/overview).
